# Migration Plan: Local Kubernetes to Google Cloud Platform (GCP)

This document outlines the necessary steps to migrate the **IoT Microservices Platform** from a local Kubernetes environment (Minikube/Docker-Desktop) to **Google Kubernetes Engine (GKE)**.

## Phase 1: Preparation & Environment Setup
- [x] **GCP Project Setup**: Created new project `iot-microservices-gcp`.
- [x] **Billing**: Ensure billing is enabled for the project (Navigate to: https://console.cloud.google.com/billing/linkedaccount?project=iot-microservices-gcp).
- [x] **CLI Configuration**: Install `gcloud` SDK, authenticate, and set project (`gcloud config set project iot-microservices-gcp`).
- [x] **Enable APIs**: 
    - `container.googleapis.com` (GKE)
    - `artifactregistry.googleapis.com` (Artifact Registry)
    - `sqladmin.googleapis.com` (Cloud SQL - if using managed DB)
- [x] **GKE Cluster Creation**:
    - Created GKE Autopilot cluster `iot-cluster` in `europe-west1`.
- [x] **Kubectl Context**: Connect `kubectl` to the new cluster:
    `gcloud container clusters get-credentials iot-cluster --region europe-west1`


## Phase 2: Docker Registry & Image Migration
- [x] **Create Artifact Registry**: Created Docker repository `iot-repo` in `europe-west1`.
- [x] **Update Image Tags**: Service tags updated and python replacement script applied.
- [x] **Push Images**: Triggered parallel async `gcloud builds submit` jobs for all microservices.
- [x] **Manifest Update**: Updated all YAML files in `manifests-k8s/prod` to point to the new Artifact Registry remote images.

## Phase 3: Infrastructure & Stateful Services
- [x] **Managed Databases (Optional but Recommended)**:
    - **MySQL**: Deployed custom `mysql-iot` container to GKE.
    - **MongoDB**: Deployed `mongo` StatefulSet to GKE.
- [x] **Storage Classes**:
    - Updated `manifests-k8s/config/pvc-k8s/` to use GKE-compatible StorageClasses (`standard-rwo`).
- [x] **RabbitMQ**:
    - Deployed RabbitMQ cluster in GKE using custom manifest.
- [x] **Data Migration**:
    - [x] Baseline test data (JSON) imported into GKE MongoDB.
    - [x] MySQL schema & initial user (Rocky) verified in GKE.
    - [x] Created `migrate.sh` for full dump/restore automation.
